# 复杂诉讼项目协作演示

> 演示 Git 驱动的律师团队 AI 协作方式

## 场景说明

这是一个关于**软件技术合作纠纷**的复杂诉讼项目。代理原告方，主张被告违反技术合作协议。

## 目录结构（12层标准）

```
litigation-ai-demo/
├── 00 - 日程管理/          # 期限、工时、任务
├── 01 - 委托材料/          # 委托合同、授权书
├── 02 - 案件分析/          # 案件概述、争议焦点
├── 03 - 法律研究/          # 法条检索、判例研究
├── 04 - 客户提供/          # 客户提供的材料
├── 05 - 证据材料/          # 证据目录、质证意见
├── 06 - 法律文书/          # 起诉状、答辩状等
├── 07 - 对方提交/          # 对方提交材料分析
├── 08 - 法院送达/          # 法院送达文书
├── 09 - 庭审笔录/          # 庭审记录
├── 10 - 综合报告/          # 案件摘要、报告
└── 11 - 参考文件/          # 参考资料、损失计算
```

## 目录职责

| 目录 | Agent | 职责 |

| `00 - 日程管理` | Scheduler | 期限管理、工时统计 |
| `01 - 委托材料` | Writer | 委托合同、授权书 |
| `02 - 案件分析` | DocAnalyzer, Strategist | 案件分析、策略 |
| `03 - 法律研究` | IssueIdentifier, Researcher | 争议焦点、法条检索 |
| `04 - 客户提供` | DocAnalyzer | 客户材料 |
| `05 - 证据材料` | EvidenceAnalyzer | 证据分析、质证 |
| `06 - 法律文书` | Writer | 起诉状、答辩状等 |
| `07 - 对方提交` | DocAnalyzer | 对方材料分析 |
| `08 - 法院送达` | DocAnalyzer, Scheduler | 传票、判决书 |
| `09 - 庭审笔录` | DocAnalyzer, Summarizer | 庭审记录 |
| `10 - 综合报告` | Summarizer, Reporter | 案件摘要、报告 |
| `11 - 参考文件` | Researcher | 参考案例、法条 |

## 协作流程

### 分支处理

```bash
# 从 main 创建研究分支
git checkout -b research/your-topic

# 完成工作后提交
git add .
git commit -m "描述本次工作"
git push origin research/your-topic
```

### PR 审核

1. 分支完成 → 创建 PR
2. 主办律师审核
3. 讨论修订
4. 合并到 main
