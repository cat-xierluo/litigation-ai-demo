# 复杂诉讼项目协作演示

> 演示 Git 驱动的律师团队 AI 协作方式

## 场景说明

这是一个关于**软件技术合作纠纷**的复杂诉讼项目。代理原告方，主张被告违反技术合作协议。

本案的特点：
- 多方向研究任务并行推进
- 大量技术文档需要梳理
- 团队多人协作，边界清晰
- AI agent 可参与各分支工作

## 目录结构

```
litigation-ai-demo/
├── 00 - 日程管理/      # 期限、工时、任务
├── 01 - 委托材料/      # 委托合同、授权书
├── 02 - 案件分析/      # 案件概述、争议焦点
├── 03 - 法律研究/      # 法条检索、判例研究
├── 04 - 客户提供/      # 客户提供的材料
├── 05 - 证据材料/      # 证据目录、质证意见
├── 06 - 法律文书/      # 起诉状、答辩状等
├── 07 - 对方提交/      # 对方材料
├── 08 - 法院送达/      # 法院文书
├── 09 - 庭审笔录/      # 庭审记录
├── 10 - 综合报告/      # 案件摘要、报告
└── 11 - 参考文件/      # 参考资料
```
