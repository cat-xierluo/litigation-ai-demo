# 项目协同规范

## 分支命名规范

- `research/<方向>` - 研究分支，如 `research/invalidity-defense`
- `docs/<主题>` - 文档分支，如 `docs/claims-revision`

## Issue 使用规范

每个研究任务创建一个 Issue，包含：
- 任务背景
- 研究目标
- 交付物要求
- 截止日期

## PR 规范

PR 应包含：
- 简要说明本次做了什么
- 交付物清单
- 待审核事项
- 关联的 Issue 编号

## 审核流程

1. 分支完成 → 创建 PR
2. 主办律师审核
3. 讨论修订
4. 合并到 main
